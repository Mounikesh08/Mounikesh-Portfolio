MediGPT – AI-Powered Healthcare Assistant

A healthcare-focused LLM assistant built to answer domain-specific medical questions using Retrieval-Augmented Generation, semantic retrieval, and trusted clinical documents.

View GitHub Live Demo

Problem

Healthcare information is often stored in PDFs and unstructured clinical documents, making it difficult for users to quickly retrieve accurate, domain-specific answers. Generic chatbots can hallucinate, which is risky in healthcare-related contexts.

Solution

Built a domain-specific RAG system that ingests clinical documents, chunks them, generates embeddings, stores them in vector databases, retrieves relevant medical context, and produces grounded responses using an LLM. The backend uses FastAPI and is deployed in a Dockerized cloud-ready setup.

Architecture

  • Clinical document ingestion
  • PDF and text preprocessing
  • Chunking for retrieval optimization
  • Embedding generation for semantic retrieval
  • Vector storage using FAISS / ChromaDB
  • Retrieval-grounded LLM response generation
  • FastAPI backend for low-latency interaction
  • Dockerized deployment on AWS-ready infrastructure

Tech Stack

Python FastAPI LangChain AWS Bedrock FAISS ChromaDB Docker AWS

Key Features

  • Domain-specific healthcare document querying
  • Retrieval grounding to reduce hallucinations
  • Low-latency API interaction
  • Semantic search for relevant clinical context
  • Evaluation workflow for answer quality and retrieval performance

Impact

  • Enabled faster access to relevant medical information from unstructured documents
  • Improved answer grounding and trustworthiness compared to generic chatbots
  • Demonstrated safe adaptation of LLM systems for domain-specific use cases
