Solar AI Operations Copilot

An end-to-end Applied AI system for solar operations, combining document-grounded RAG, warranty/support triage, solar generation forecasting, monitoring, feedback loops, and responsible AI guardrails.

View GitHub Live Demo

Problem

Solar companies and support teams handle large volumes of warranty policies, installation documents, customer issue reports, battery concerns, and performance questions. Without AI assistance, support teams spend significant time searching documents, identifying warranty requirements, triaging issues, and estimating system performance under changing weather and battery conditions.

Solution

Built a production-style Solar AI Operations Copilot that allows users to upload solar documents, ask policy-grounded questions, submit warranty/support issues, and estimate solar generation using machine learning. The system includes a FastAPI backend, Streamlit customer UI, RAG pipeline, warranty triage logic, forecasting model, monitoring layer, feedback capture, tests, CI/CD, Docker readiness, and responsible AI guardrails.

Architecture

  • Customer-facing Streamlit frontend for document Q&A, support issue review, and solar generation estimation
  • FastAPI backend exposing modular endpoints for RAG, triage, forecasting, monitoring, and guardrails
  • PDF ingestion pipeline with text extraction, chunking, embeddings, ChromaDB storage, and citation-based retrieval
  • Warranty/support triage agent for issue classification, priority assignment, escalation, and required document guidance
  • Solar generation forecasting module using structured weather, irradiance, battery SOC, and calendar features
  • SQLite-based monitoring and feedback layer for latency, confidence, status, errors, and user feedback tracking
  • Responsible AI guardrails for missing citations, low confidence, risky warranty claims, invalid inputs, and human review recommendations

Tech Stack

Python FastAPI Streamlit LangChain ChromaDB Hugging Face Scikit-learn Random Forest SQLite Pytest Docker GitHub Actions

Key Features

  • Solar document RAG assistant with PDF upload, embeddings, retrieval, and source citations
  • Warranty and support triage agent with issue category, priority, escalation decision, and required documents
  • Solar + battery generation estimator using machine learning forecasting on weather and battery features
  • Customer-friendly Streamlit interface for document questions, support issues, and generation estimates
  • Monitoring backend that logs module usage, request status, latency, confidence scores, and errors
  • User feedback loop with Helpful / Not Helpful responses and optional comments
  • Responsible AI guardrails for citation validation, low-confidence warnings, risky language detection, and human review
  • Production-readiness setup with tests, Dockerfile, GitHub Actions CI/CD, .env.example, and README documentation

Impact

  • Demonstrates a real-world Applied AI workflow aligned with solar, clean energy, warranty, and operations use cases
  • Reduces manual document searching by enabling citation-grounded answers from uploaded solar policies
  • Improves support consistency by standardizing issue triage, required documents, and escalation recommendations
  • Shows practical machine learning deployment through a solar generation forecasting API and customer UI
  • Adds production-style reliability through monitoring, feedback capture, guardrails, automated tests, and CI/CD
  • Highlights end-to-end AI engineering skills across GenAI, RAG, ML, FastAPI, Streamlit, DevOps, and responsible AI
