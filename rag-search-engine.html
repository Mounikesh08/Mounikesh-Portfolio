RAG Document Search Engine – AWS + LangChain

A scalable semantic document search platform that enables natural language querying over unstructured documents using vector retrieval and LLM-powered grounded answers.

View GitHub Live Demo

Problem

Large collections of unstructured documents are difficult to search effectively using traditional keyword-based systems. Users often know what they want conceptually, but not the exact terms to search, which leads to wasted time and poor retrieval quality.

Solution

Built a semantic search engine using LangChain, AWS Bedrock, FAISS, and Streamlit. The system converts documents into embeddings, retrieves the most relevant chunks for a query, and uses an LLM to generate grounded answers in natural language.

Architecture

  • Document ingestion and preprocessing
  • Chunking pipeline for retrieval optimization
  • Embedding generation and vector indexing
  • FAISS-based semantic retrieval
  • AWS Bedrock integration for response generation
  • Streamlit front end for interactive querying

Tech Stack

Python LangChain AWS Bedrock FAISS Streamlit

Key Features

  • Natural language querying over unstructured documents
  • Semantic search instead of keyword-only retrieval
  • Interactive UI for non-technical users
  • Prompt engineering to improve answer quality
  • Optimized retrieval for latency and relevance

Impact

  • Reduced friction in exploring large document collections
  • Improved accessibility for non-technical users through a simple interface
  • Demonstrated real-time grounded document Q&A with better retrieval quality
